57   * <strong>1. How to use catch-exception?</strong>
58   *
59   * The most basic usage is:
60   * <code>import static com.googlecode.catchexception.CatchException.*;
61   *
62   * // call customerService.prepareBilling(Prize.Zero)
63   * // and catch the exception if any is thrown
64   * catchException(customerService).prepareBilling(Prize.Zero);
65   *
66   * // assert that an IllegalArgumentException was thrown
67   * assert caughtException() instanceof IllegalArgumentException;</code>
68   *
69   * You can combine the two lines of code in a single one if you like:
70   * <code>// call customerService.prepareBilling(Prize.Zero)
71   * // and throw an ExceptionNotThrownAssertionError if
72   * // the expected exception is not thrown
73   * verifyException(customerService, IllegalArgumentException.class).prepareBilling(Prize.Zero);</code>
74   * There is a minor difference between both variants. In the first variant you
75   * must start the JVM with option <code>-ea</code> to enable the assertion. The
76   * second variant does not use JDK assertions and ,therefore, always verifies
77   * the caught exception.
78   *
79   * A third variant allows you to select the type of exceptions you want to catch
80   * (no verification involved):
81   * <code>// catch IllegalArgumentExceptions but no other exceptions
82   * catchException(customerService, IllegalArgumentException.class).prepareBilling(Prize.Zero);</code>
83   *
84   * The fourth and last variant verifies that some exception is thrown, i.e. the
85   * type of the exception does not matter:
86   * <code>verifyException(customerService).prepareBilling(Prize.Zero);</code>
87   *
88   * In all variants you can use <code>caughtException()</code> afterwards to
89   * inspect the caught exception.
90   *
91   * Finally, there some alternative ways to catch and verify exceptions:
92   * <ul>
93   * <li>{@link BDDCatchException} - a BDD-like approach,
94   * <li> {@link CatchExceptionHamcrestMatchers} - Hamcrest assertions
95   * </ul>
96   * <strong>2. What is this stuff actually good for?</strong>
97   *
98   * This class targets concise and robust code in tests. Dadid Saff, a commiter
99   * to JUnit, has <a
100  * href="http://shareandenjoy.saff.net/2006/12/assertthrownexception_20.html"
101  * >discussed</a> this approach in 2007. Let me summarize the arguments here.
102  *
103  * There are two advantages of the approach proposed here in comparison to the
104  * use of try/catch blocks.
105  * <ul>
106  * <li>The test is more concise and easier to read.
107  * <li>The test cannot be corrupted by a missing assertion. Assume you forgot to
108  * type <code>fail()</code> behind the method call that is expected to throw an
109  * exception.
110  * </ul>
111  *
112  * There are also some advantages of this approach in comparison to test
113  * runner-specific mechanisms that catch and verify exceptions.
114  * <ul>
115  * <li>A single test can verify more than one thrown exception.
116  * <li>The test can verify the properties of the thrown exception after the
117  * exception is caught.
118  * <li>The test can specify by which method call the exception must be thrown.
119  * <li>The test does not depend on a specific test runner (JUnit4, TestNG).
120  * </ul>
121  *
122  * <strong>3. How does it work internally?</strong>
123  *
124  * The method <code>catchException(obj)</code> wraps the given object with a
125  * proxy that catches the exception, then (optionally) verifies the exception,
126  * and finally attaches the exception to the current <a
127  * id="threadlocal">thread</a> for further analysis. The <a
128  * href="#proxies">known limitations</a> for proxies apply.
129  *
130  * Is both memory consumption and runtime a concern for you? Then use try/catch
131  * blocks instead of this class. Because in this case the creation of proxies is
132  * an unnecessary overhead. If only either memory consumption or runtime is an
133  * issue for you, feel free to configure the cache of the underlying proxy
134  * factories as appropriate.
135  * <strong>4. When is the caught exception reset?</strong>
136  *
137  * The Method {@link #caughtException()} returns the exception thrown by the
138  * last method call on a proxied object in the current thread, i.e. it is reset
139  * by calling a method on the proxied object. If the called method has not
140  * thrown an exception, <code>caughtException()</code> returns null.
141  *
142  * To reset the caught exception manually, call {@link #resetCaughtException()}.
143  * At the moment there is no way to reset exceptions that have been caught in
144  * other threads.
145  * <strong>5. My code throws a ClassCastException. Why?</strong>
146  *
147  * Example:
148  * <code>StringBuilder sb = new StringBuilder();
149  * catchException(sb).charAt(-2); // throws ClassCastException</code>
150  *
151  * Probably you have tested a final class. Proxy factories usually try to
152  * subclass the type of the proxied object. This is not possible if the original
153  * class is final. But there is a way out. If the tested method belongs to an
154  * interface, then you can cast the argument (here: <code>sb</code>) to that
155  * interface or ,easier, change the declared type of the argument to the
156  * interface type. This works because the created proxy is not longer required
157  * to have the same type as the original class but it must only have the same
158  * interface.
159  * <code>// first variant
160  * StringBuilder sb = new StringBuilder();
161  * catchException((CharSequence) sb).charAt(-2); // works fine
162  *
163  * // second variant
164  * CharSequence sb = new StringBuilder();
165  * catchException(sb).charAt(-2); // works fine</code> If the tested
166  * method does no belong to an interface fall back to the try/catch-blocks or
167  * use <a
168  * href="http://code.google.com/p/catch-exception/wiki/Dependencies">Powermock
169  * </a>.
170  * <code>// example for
171  * PowerMock with JUnit4
172  * &#064;RunWith(PowerMockRunner.class)
173  * &#064;PrepareForTest({ MyFinalType.class })
174  * public class MyTest {
175  * </code>
176  * <strong>6. Do I have to care about memory leaks?</strong>
177  *
178  * This library uses a {@link ThreadLocal}. ThreadLocals are known to cause
179  * memory leaks if they refer to a class the garbage collector would like to
180  * collect. If you use this library only for testing, then memory leaks do not
181  * worry you. If you use this library for other purposes than testing, you
182  * should care.
183  *
184  * <strong>7. The caught exception is not available in another thread. Why?</strong>
185  *
186  * The caught exception is saved <a href="#threadlocal">at the thread</a> the
187  * exception is thrown in. This is the reason the exception is not visible
188  * within any other thread.
189  * <strong>8. Is there a way to get rid of the throws clause in my test
190  * method?</strong>
191  *
192  * Example:
193  * <code>public void testSomething() throws Exception {
194  * ...
195  * catchException(obj).do(); // do() throws a checked exception</code> No,
196  * although the exception is always caught you cannot omit the throws clause in
197  * your test method.
198  * <strong>11. Can I catch errors instead of exceptions?</strong>
199  *
200  * Yes, have a look at
201  * {@code com.googlecode.catchexception.throwable.CatchThrowable} (in module
202  * catch-throwable).
203  *
204  * @author rwoo
205  * @since 16.09.2011
206  */

209     /**
210      * Returns the exception caught during the last call in the current thread.
211      *
212      * @param <E> This type parameter makes some type casts redundant.
213      * @return Returns the exception caught during the last call in the current
214      * thread - if the call was made through a proxy that has been created via
215      * {@link #verifyException(ThrowingCallable, Class) verifyException()} or
216      * {@link #catchException(ThrowingCallable, Class) catchException()}. Returns
217      * null when no exception was caught.
218      */
219     public static <E extends Exception> E caughtException() {
220         return ExceptionHolder.get();
221     }

227     /**
228      * Use it to verify that an exception is thrown and to get access to the
229      * thrown exception (for further verifications).
230      *
231      * The following example verifies that obj.doX() throws a Exception:
232      * <code>verifyException(obj).doX(); // catch and verify
233      * assert "foobar".equals(caughtException().getMessage()); // further analysis
234      * </code>
235      *
236      * If <code>doX()</code> does not throw a <code>Exception</code>, then a
237      * {@link ExceptionNotThrownAssertionError} is thrown. Otherwise the thrown
238      * exception can be retrieved via {@link #caughtException()}.
239      *
240      *
241      * @param actor The instance that shall be proxied. Must not be
242      *              <code>null</code>.
243      */
244     public static void verifyException(ThrowingCallable actor) {
245         verifyException(actor, Exception.class);
246     }
247 
248     /**
249      * Use it to verify that an exception of specific type is thrown and to get
250      * access to the thrown exception (for further verifications).
251      *
252      * The following example verifies that obj.doX() throws a MyException:
253      * <code>verifyException(obj, MyException.class).doX(); // catch and verify
254      * assert "foobar".equals(caughtException().getMessage()); // further analysis
255      * </code>
256      *
257      * If <code>doX()</code> does not throw a <code>MyException</code>, then a
258      * {@link ExceptionNotThrownAssertionError} is thrown. Otherwise the thrown
259      * exception can be retrieved via {@link #caughtException()}.
260      *
261      *
262      * @param actor The instance that shall be proxied. Must not be
263      *              <code>null</code>.
264      * @param clazz The type of the exception that shall be thrown by the
265      *              underlying object. Must not be <code>null</code>.
266      */
267     public static void verifyException(ThrowingCallable actor, Class<? extends Exception> clazz) {
268         validateArguments(actor, clazz);
269         catchException(actor, clazz, true);
270     }
271 
272     /**
273      * Use it to catch an exception and to get access to the thrown exception
274      * (for further verifications).
275      *
276      * In the following example you catch exceptions that are thrown by
277      * obj.doX():
278      * <code>catchException(obj).doX(); // catch
279      * if (caughtException() != null) {
280      * assert "foobar".equals(caughtException().getMessage()); // further analysis
281      * }</code>
282      * If <code>doX()</code> throws a exception, then {@link #caughtException()}
283      * will return the caught exception. If <code>doX()</code> does not throw a
284      * exception, then {@link #caughtException()} will return <code>null</code>.
285      *
286      *
287      * @param actor The instance that shall be proxied. Must not be
288      *              <code>null</code>.
289      */
290     public static void catchException(com.googlecode.catchexception.ThrowingCallable actor) {
291         validateArguments(actor, Exception.class);
292         catchException(actor, Exception.class, false);
293     }
294 
295     /**
296      * Use it to catch an exception of a specific type and to get access to the
297      * thrown exception (for further verifications).
298      *
299      * In the following example you catch exceptions of type MyException that
300      * are thrown by obj.doX():
301      * <code>catchException(obj, MyException.class).doX(); // catch
302      * if (caughtException() != null) {
303      * assert "foobar".equals(caughtException().getMessage()); // further analysis
304      * }</code>
305      * If <code>doX()</code> throws a <code>MyException</code>, then
306      * {@link #caughtException()} will return the caught exception. If
307      * <code>doX()</code> does not throw a <code>MyException</code>, then
308      * {@link #caughtException()} will return <code>null</code>. If
309      * <code>doX()</code> throws an exception of another type, i.e. not a
310      * subclass but another class, then this exception is not thrown and
311      * {@link #caughtException()} will return <code>null</code>.
312      *
313      *
314      * @param actor The instance that shall be proxied. Must not be
315      *              <code>null</code>.
316      * @param clazz The type of the exception that shall be caught. Must not be
317      *              <code>null</code>.
318      */
319     public static void catchException(ThrowingCallable actor, Class<? extends Exception> clazz) {
320         validateArguments(actor, clazz);
321         catchException(actor, clazz, false);
322     }
323 
324     private static void catchException(ThrowingCallable actor, Class<? extends Exception> clazz,
325                                        boolean assertException) {
326         resetCaughtException();
327         Exception exception = ExceptionCaptor.captureThrowable(actor);
328         if (exception == null) {
329             if (!assertException) {
330                 return;
331             } else {
332                 throw new ExceptionNotThrownAssertionError(clazz);
333             }
334         }
335         // is the thrown exception of the expected type?
336         if (clazz.isAssignableFrom(exception.getClass())) {
337             ExceptionHolder.set(exception);
338         } else {
339             if (assertException) {
340                 throw new ExceptionNotThrownAssertionError(clazz, exception);
341             } else {
342                 ExceptionUtil.sneakyThrow(exception);
343             }
344         }
345     }
346 
347     private static void validateArguments(ThrowingCallable actor, Class<? extends Exception> clazz) {
348         if (actor == null) throw new IllegalArgumentException("obj must not be null");
349         if (clazz == null) throw new IllegalArgumentException("exceptionClazz must not be null");
350     }
351 
352     /**
353      * Sets the {@link #caughtException() caught exception} to null. This does
354      * not affect exceptions saved at threads other than the current one.
355      *
356      * Actually you probably never need to call this method because each method
357      * call on a proxied object in the current thread resets the caught
358      * exception. But if you want to improve test isolation or if you want to
359      * 'clean up' after testing (to avoid memory leaks), call the method before
360      * or after testing.
361      */
362     public static void resetCaughtException() {
363         ExceptionHolder.set(null);
364     }
